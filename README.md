# 🚀 Java CI/CD Pipeline using Jenkins, Gradle & Docker

## 📌 Project Overview
This project demonstrates a complete CI/CD pipeline for a Java application using Jenkins, Gradle, and Docker.  
The pipeline automates the process of fetching code from GitHub, building the application, and running it inside a Docker container.

The main goal of this project is to understand how DevOps tools work together to automate software delivery.

---

## ⚙️ Tools & Technologies Used
- Java
- Gradle
- Jenkins
- Docker
- Git & GitHub
- Linux (Jenkins container environment)

---

## 🔄 CI/CD Pipeline Flow
1. Jenkins pulls the latest code from GitHub repository.
2. Gradle builds the Java application.
3. Docker builds an image for the application.
4. The application runs inside a Docker container.
5. Jenkins manages the full automation process.

---

## 🚀 How It Works
- The pipeline is triggered manually or automatically from Jenkins.
- Source code is fetched from GitHub.
- The project is built using Gradle Wrapper.
- A Docker image is created from the build output.
- The container is executed automatically.
